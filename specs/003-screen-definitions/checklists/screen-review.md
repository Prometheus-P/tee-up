# Screen Review Checklist (화면 검토 체크리스트)

이 체크리스트는 화면 정의서 작성 및 구현 검토에 사용합니다.

---

## 1. 화면 정의서 완성도 검토

### 필수 섹션 포함 여부

- [ ] **화면 개요** - 목적, 사용자 흐름, 접근 조건
- [ ] **레이아웃 구조** - 텍스트 와이어프레임
- [ ] **컴포넌트 목록** - 컴포넌트별 상세 정의
- [ ] **데이터 요구사항** - Server Actions, 스키마
- [ ] **인터랙션 정의** - Given-When-Then 시나리오
- [ ] **접근성 요구사항** - 키보드, 스크린 리더
- [ ] **에러 케이스** - 에러 유형별 처리 방법
- [ ] **관련 화면** - 이전/다음/연관 화면 링크

### 메타데이터 확인

- [ ] Route 정확성
- [ ] Status (Implemented / Not Implemented)
- [ ] Priority (P0 / P1 / P2 / P3)
- [ ] Dependencies 명시
- [ ] Last Updated 날짜

---

## 2. 디자인 시스템 준수 검토

### 색상 토큰 사용

- [ ] 모든 색상이 `--tee-*` 토큰 사용
- [ ] 하드코딩된 색상값 없음
- [ ] 다크 모드 고려됨

### 타이포그래피

- [ ] Heading 계층 (h1 > h2 > h3) 준수
- [ ] 본문 텍스트 적절한 크기 사용
- [ ] 폰트 토큰 사용

### 간격 (Spacing)

- [ ] Tailwind space-* 클래스 사용
- [ ] 일관된 패딩/마진

### 컴포넌트

- [ ] shadcn/ui 컴포넌트 우선 사용
- [ ] 커스텀 컴포넌트 필요시 문서화
- [ ] 컴포넌트 상태 정의 (hover, focus, active, disabled)

---

## 3. 접근성 (A11y) 검토

### 키보드 네비게이션

- [ ] 모든 인터랙티브 요소 Tab으로 접근 가능
- [ ] 논리적인 탭 순서
- [ ] 포커스 표시자 명확
- [ ] Escape로 모달/드롭다운 닫기

### 스크린 리더

- [ ] 적절한 ARIA 레이블
- [ ] 동적 콘텐츠에 aria-live 적용
- [ ] 이미지에 alt 텍스트
- [ ] 폼 필드에 label 연결

### 색상 대비

- [ ] 텍스트 대비 4.5:1 이상
- [ ] 큰 텍스트 대비 3:1 이상
- [ ] 색상만으로 정보 전달하지 않음

### 폼

- [ ] 필수 필드 명시 (aria-required)
- [ ] 에러 메시지 연결 (aria-describedby)
- [ ] 자동완성 속성 적용

---

## 4. 반응형 디자인 검토

### 브레이크포인트

- [ ] Mobile (< 640px) 레이아웃 정의
- [ ] Tablet (640px ~ 1024px) 레이아웃 정의
- [ ] Desktop (> 1024px) 레이아웃 정의

### 모바일 우선

- [ ] 기본 스타일이 모바일 기준
- [ ] 점진적 향상 (sm: → md: → lg:)
- [ ] 터치 타겟 44x44px 이상

---

## 5. 데이터 흐름 검토

### Server Actions

- [ ] 모든 Server Action 명시
- [ ] 입력/출력 타입 정의
- [ ] 에러 처리 방법 정의

### 상태 관리

- [ ] 로딩 상태 처리
- [ ] 에러 상태 처리
- [ ] 빈 상태 처리

### 캐싱 & 재검증

- [ ] revalidatePath 적용 여부
- [ ] 실시간 업데이트 필요 여부

---

## 6. 보안 검토

### 인증 & 권한

- [ ] 접근 조건 명확히 정의
- [ ] 권한 없는 접근 시 처리 방법
- [ ] 민감한 데이터 마스킹

### 입력 검증

- [ ] 클라이언트 검증
- [ ] 서버 검증 (Server Action에서)
- [ ] XSS 방지

---

## 7. 성능 검토

### 이미지

- [ ] Next.js Image 컴포넌트 사용
- [ ] 적절한 크기/포맷
- [ ] Lazy loading 적용

### 데이터 페칭

- [ ] 불필요한 데이터 요청 없음
- [ ] 페이지네이션/무한 스크롤 적용
- [ ] Skeleton UI 적용

---

## 8. 화면 간 일관성 검토

### 네비게이션

- [ ] 헤더/푸터 일관성
- [ ] 뒤로 가기 동작 명확
- [ ] 브레드크럼 적용 (필요시)

### UI 패턴

- [ ] 버튼 스타일 일관성
- [ ] 모달/토스트 일관성
- [ ] 테이블/카드 스타일 일관성

### 용어

- [ ] 동일한 개념에 동일한 용어 사용
- [ ] 한글/영어 혼용 규칙 준수

---

## 9. 미구현 화면 특별 검토

미구현 화면(`Status: Not Implemented`)의 경우 추가 검토:

- [ ] 구현 가이드 (코드 예시) 포함
- [ ] 필요한 Server Action 목록
- [ ] 필요한 컴포넌트 목록
- [ ] 데이터베이스 스키마 변경 필요 여부
- [ ] 우선순위 적절성

---

## 10. 최종 검토

### 문서 품질

- [ ] 오타 없음
- [ ] Markdown 포맷 정상
- [ ] 링크 유효성 확인
- [ ] 다이어그램 가독성

### 추적성

- [ ] 요구사항과 연결됨
- [ ] 디자인 시스템 문서 참조
- [ ] 관련 화면 링크 유효
