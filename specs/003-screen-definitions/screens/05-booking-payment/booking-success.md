# Screen: Booking Success (예약 완료)

**Route**: `/booking/success`
**Status**: Implemented
**Last Updated**: 2025-12-18
**Priority**: P1
**Dependencies**: Booking System

---

## 1. 화면 개요

### 목적
레슨 예약이 성공적으로 완료되었음을 알리고, 다음 단계 안내를 제공한다.

### 사용자 흐름
```
프로 포트폴리오 → 예약하기 → 예약 정보 입력 → /booking/success
```

### 접근 조건
- **인증 필요**: No
- **권한 수준**: guest
- **전제 조건**: 유효한 예약 완료

## 3. 주요 컴포넌트

### 3.1 SuccessIcon

**성공 아이콘**

| Property | Value |
|----------|-------|
| 아이콘 | 체크마크 (✓) 원형 |
| 크기 | 64px |
| 색상 | `--tee-success` (#388E3C) |
| 애니메이션 | Scale-in + 체크 드로잉 |

### 3.2 BookingInfoCard

**예약 정보 카드**

| Field | Description |
|-------|-------------|
| 프로 | 프로 이름 + 타이틀 |
| 날짜 | 예약 날짜 (요일 포함) |
| 시간 | 레슨 시작 시간 |
| 장소 | 레슨 장소 |
| 예약 번호 | 고유 예약 ID |

### 3.3 NoticeCard

**안내 카드**

| Element | Description |
|---------|-------------|
| 제목 | "예약 안내" |
| 항목 | 도착 시간, 취소 정책, 문의 안내 |
| 스타일 | `--tee-surface` 배경, 불릿 리스트 |

### 3.4 ActionButtons

**액션 버튼**

| Button | Style | Action |
|--------|-------|--------|
| 캘린더에 추가 | Secondary | 캘린더 앱 연동 |
| 홈으로 | Primary | `/` 이동 |

## 4. 데이터 요구사항

### URL Parameters

| Param | Description |
|-------|-------------|
| `bookingId` | 예약 ID |

### Server Actions

| Action | 설명 |
|--------|------|
| `getBookingDetails` | 예약 상세 정보 조회 |

### 데이터 스키마

```typescript
interface BookingDetails {
  id: string;
  booking_number: string;
  pro_name: string;
  pro_title: string;
  date: string;
  time: string;
  location: string;
  status: 'confirmed' | 'pending';
  created_at: string;
}
```

## 5. 인터랙션 정의

### 5.1 캘린더 추가

1. **Given** 예약 완료 화면이 표시될 때
2. **When** "캘린더에 추가" 버튼을 클릭하면
3. **Then** .ics 파일 다운로드 또는 캘린더 앱 선택 메뉴 표시

### 5.2 페이지 로드 애니메이션

1. **Given** 페이지가 로드될 때
2. **Then** 성공 아이콘이 scale-in 애니메이션으로 나타남
3. **Then** 체크마크가 드로잉 애니메이션으로 그려짐
4. **Then** 정보 카드가 fade-up으로 나타남
