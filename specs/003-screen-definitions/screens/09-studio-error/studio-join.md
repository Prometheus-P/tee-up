# Screen: Studio Join (스튜디오 가입)

**Route**: `/studio/join/[code]`
**Status**: Implemented
**Last Updated**: 2025-12-18
**Priority**: P2
**Dependencies**: Auth, Studios System

---

## 1. 화면 개요

### 목적
초대 링크를 통해 스튜디오에 가입하는 페이지.

### 사용자 흐름
```
초대 링크 클릭 → /studio/join/[code] → 로그인/가입 → 스튜디오 가입 완료
```

### 접근 조건
- **인증 필요**: Yes (가입 시)
- **권한 수준**: pro
- **전제 조건**: 유효한 초대 코드

## 3. 주요 컴포넌트

### 3.1 StudioPreview

**스튜디오 미리보기**

| Element | Description |
|---------|-------------|
| 로고 | 스튜디오 로고 이미지 |
| 이름 | 스튜디오 이름 |
| 위치 | 스튜디오 위치 (선택) |

### 3.2 InviteMessage

**초대 메시지**

| Element | Description |
|---------|-------------|
| 제목 | "{스튜디오명}에서 초대했습니다" |
| 설명 | 가입 안내 또는 혜택 설명 |

### 3.3 JoinActions

**액션 버튼**

| State | Buttons |
|-------|---------|
| 비로그인 | 로그인하기 (Primary) |
| 로그인 | 취소 (Secondary), 가입하기 (Primary) |

## 4. 데이터 요구사항

### URL Parameters

| Param | Description |
|-------|-------------|
| `code` | 초대 코드 |

### Server Actions

| Action | 설명 |
|--------|------|
| `getStudioByInviteCode` | 초대 코드로 스튜디오 조회 |
| `joinStudioByCode` | 초대 코드로 가입 |

---

## 5. 인터랙션 정의

### 5.1 로그인 후 가입

1. **Given** 비로그인 상태로 접근할 때
2. **When** "로그인하기" 버튼을 클릭하면
3. **Then** `/auth/login?redirect=/studio/join/[code]`로 이동
4. **When** 로그인 완료 후
5. **Then** 가입 확인 화면으로 돌아온다

### 5.2 가입 완료

1. **Given** 로그인 상태로 "가입하기"를 클릭하면
2. **Then** 스튜디오 멤버로 추가된다
3. **Then** `/dashboard/studio`로 리디렉션

---

## 6. 에러 케이스

| 에러 유형 | 처리 |
|----------|------|
| 잘못된 초대 코드 | [Studio Join Invalid](./studio-join-invalid.md)로 이동 |
| 이미 가입됨 | "이미 가입된 스튜디오입니다" 메시지 |
| 프로 승인 필요 | "프로 승인 후 가입 가능합니다" 메시지 |
